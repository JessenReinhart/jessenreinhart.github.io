How it works

Three steps. Zero setup.

No accounts, no pairing, no app store. If your phone is on the same Wi-Fi, it just works.

01 / RUN

Run

Run share-term in any active project directory. It auto-detects log files, tmux panes, or spawns a live PTY.

$ share-term
02 / SCAN

Scan

Aim your phone camera at the QR code printed in your terminal. The PWA receiver opens instantly — nothing to download.

📷 scan → open
03 / STREAM

Stream

Watch live logs with smart filtering, text search, and error alerts. Your phone vibrates the moment a build fails.

ws:// <lan-ip>:8080
Features

Built like a developer tool should be.

Precise, lightning-fast, zero bloat. Every byte streamed over a raw WebSocket — no heavy runtime, no telemetry.

Zero-config DX

No App Store. Opens as a PWA.

The receiver is a single responsive HTML file served from the CLI itself. Your phone installs it to the home screen in one tap and launches straight into a full terminal — xterm.js rendered locally, with cursor movement, colors, and layout intact.

ANSI Faithful

Real SGR, 256-color aware. Logs render exactly like your terminal.

Smart Notifications

Phone vibrates the exact moment a build fails or an exception is thrown.

Battery Optimized

Pure WebSocket data stream. Ultra-low battery, negligible CPU on the laptop.

Live Terminal Sharing

Spawns a shared shell via node-pty (ConPTY / openpty) — your phone sees it live with full ANSI colors. No tmux required, but tmux attach is there when you want it.

Wide-TUI Aware

PTY sources report real width, so the phone renders at the true column count — box-drawing art stays crisp instead of wrapping.
